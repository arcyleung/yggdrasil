Shareable overview

Yggdrasil

MCP-native, org-wide trajectory experience memory for autonomous agents—so fleets can reuse what was tried, what it cost, and who to talk to next.

PoCtenant: labnot document RAG

Single-file deck · ← / → or buttons · no network required after download

Motivation

Agent fleets create context debt faster than humans can document

What breaks today

  • Long-horizon agent work is mostly invisible after the chat ends
  • Wikis lag reality; nobody wants to maintain AI-generated docs
  • The next agent (or human) re-pays research, setup, and failure cost
  • Handoff is tribal: “ask Alex” with no searchable evidence

What we need instead

  • Memory of strategies executed, not just chat snippets or facts
  • Retrieval keyed by current task + scaffold
  • Full traces on demand (steps, artifacts, effort)
  • First-class owner / agent_id for human follow-up
Problem framing

Humans see ~20% of agent context at test time

For audit and strategy—not just the final answer—you need the whole trajectory: paths, retries, tools, outcomes, waste.

  • Unit of memory: trajectory = task + scaffold + steps + progress + outcome + effort + artifacts
  • Search key: embed only current task + scaffold (history lives in steps)
  • Org scope: one lab retrieval plane; writes still bind to a real owner server-side
Value

Four bets that pay for the system

1. No doc maintenance burden

Trajectories are the experiential source of truth—attested by execution, not a wiki someone was supposed to update.

2. Tombstone memoization

Success and failure outcomes are cheap to consult, expensive to rediscover. High failure_waste_seconds is a warning, not a template.

3. Strategy-level understanding

Search at strategy grain; get_trajectory for full method when shortlisted—not always-on transcript RAG.

4. Security & management leverage

Semantic search over what agents actually ran; staff by who has done this before (owner on every useful hit).

Differentiation

Not Mem0 / chat memory / doc RAG

Those optimize “what does the agent know?” Yggdrasil optimizes “what strategies did we execute, at what cost, producing what, and who owns follow-up?”

2. Search

3. Deepen

4. Write

Architecture overview

Control plane vs data plane

Control plane
FastAPI UI · lab login · personalized skill.md / mcp.json · token exchange · Streamable HTTP MCP gateway · optional chat · often Tailscale / Funnel /ygg
Data plane
SQLite SoT trajectories + steps · Qdrant multi-aspect vectors (task / scaffold) · embed endpoint · MCP tools enforce Principal (tenant, owner, scopes)
Ingress
Live agent writes via MCP · optional hydration from Mongo proxy logs → segment → embed (experience_grade vs archive gates)
Tenancy
lab = shared org retrieval · demo = external trial · mandatory tenant_id on vector search
Architecture · seams

Ports that matter when you scale

TrajectoryStore

SQLite today → Postgres (or equivalent) for multi-replica SoT without rewriting MCP tools.

VectorIndex

Qdrant client; tenant filter on every query; replica-safe point ids.

Embedder

Task/scaffold aspects only; batch, timeouts, circuit breaker; async worker / outbox as N grows.

TokenStore + Principal

Bearer ygg_… binds owner/tenant; writes cannot spoof attribution.

Search path: Qdrant filter + vectors → hydrate trajectories from SoT by id (not full table scans).

Retrieval trust

Gates so “memory” is not archive noise

  • agent mode (default): stricter strategy recall; exclude hydration/archive provenance tags
  • lab mode: wide org forensics; prefers experience-grade memory—still does not launder archive trajectories
  • Empty gated results are OK: “no trusted prior experience”
  • Ranked table uses ISO event time at; agents bin today / week / month client-side (avoid embed-time updated_at false “today”)
Rollout for the lab

Humans don’t run the stack—they paste a skill

  1. Open control plane URL · lab login with mapped sk-… key
  2. Download skill (MCP URL + bearer policies in one document)
  3. Install skill in Codex / Cursor / Grok / … and point MCP client at host
  4. Ask org questions: “who set up X?” → search → name owners → optional deep trajectory

Operators: uvicorn yggdrasil.web.app + Qdrant + embed + user_mapping.yaml · restart control plane after code changes (payload schema drift breaks search).

Takeaway

Org experience memory, MCP-shaped

Yggdrasil turns agent work into searchable strategies with attribution—so the lab stops rediscovering the same pain and starts compounding what already worked.

  • Search before uncertain / high-overhead work
  • Surface owners for human follow-up
  • Write non-trivial runs; segment long sessions; finalize outcomes + effort
